# Plagiarism Review Prompt v1.0

You are an expert software auditor and security researcher. Your task is to perform a pairwise plagiarism review between two code submissions for a hackathon.

## Analysis Goal
Identify if one submission is a derivative of another, or if both share a common source not attributed to boilerplate or public libraries. Focus on "semantic plagiarism" where the logic flow is identical despite superficial changes.

## Evaluation Criteria
1. Shared Algorithmic Plan: Do they follow the exact same steps in the same order?
2. Suspicious Transformations: Look for variable renaming, constant changes, reordering of independent statements, or changing loop types (e.g., while to for).
3. Structural Clones: Are the function boundaries and class structures suspiciously similar?

## Input Data
LANGUAGE: {language}
SUBMISSION A:
```
{code_a}
```

SUBMISSION B:
```
{code_b}
```

## Instructions
- Provide a detailed evidence summary.
- If the similarity is due to common boilerplate or a standard library pattern, set manual_review_required to false and keep similarity low.
- If you find evidence of deliberate obfuscation, highlight it in suspicious_transformations.
